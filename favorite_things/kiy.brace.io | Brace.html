What is Shipping?

There are two versions of your site

Draft-version updates automatically as you make changes to the files in Dropbox. It has a different URL than your actual website

Production-version is the website behind your site's URL. You can update it by shipping.

Shipping copies the contents of draft version into production version. Note that it might take some time for the shipped site to update.

Why two versions?

With two versions you can try things out in draft mode without affecting the production site. Your draft version updates automatically and has its own address that you can share. This way you can be bold, get crazy, and know that your production site is safe.

What's a Public Folder?

In most cases, you don't have to worry about public folder.

Public Folder is the folder we'll look into inside the designated Brace site folder () for your files. By default, it's just the root ("/"), so when you have index.html there, it works as expected.

However, you can change it to be any subfolder there. This might be useful for example if you are using some generator like Jekyll to build your site. If you change the public folder to be "/_site", then we'll render "/_site/index.html" when you visit your site.

Password protected sites

You can set a password for Draft and/or Production sites. If it's set, anyone visiting the site is first redirected to a view prompting for a password. With this, you can for example make sure site visitors can't go to draft version and check out your upcoming new stuff.
